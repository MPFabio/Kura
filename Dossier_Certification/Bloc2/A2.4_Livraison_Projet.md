## 1. Organisation et réalisation des tests

### Périmètre des tests

#### Types de tests

**Tests unitaires :**
- **Périmètre :** Fonctions, méthodes individuelles
- **Objectif :** Vérifier le comportement isolé
- **Couverture cible :** > 80%
- **Outils :** Go testing, Jest (React)

**Tests d'intégration :**
- **Périmètre :** Interactions entre composants, services
- **Objectif :** Vérifier l'intégration
- **Couverture cible :** > 70%
- **Outils :** Go testing, Supertest (React)

**Tests end-to-end (e2e) :**
- **Périmètre :** Scénarios utilisateur complets
- **Objectif :** Vérifier le fonctionnement de bout en bout
- **Couverture cible :** Scénarios critiques (100%)
- **Outils :** Cypress, Playwright

**Tests de performance :**
- **Périmètre :** Temps de réponse, charge, scalabilité
- **Objectif :** Vérifier les performances
- **Seuils :** < 200ms (p95), support 100+ clusters
- **Outils :** k6, Apache JMeter

**Tests de sécurité :**
- **Périmètre :** Vulnérabilités, authentification, autorisation
- **Objectif :** Vérifier la sécurité
- **Seuils :** 0 vulnérabilité critique
- **Outils :** Snyk, OWASP ZAP

### Exemple de test pour s'assurer de la conformité

#### Test : Authentification et autorisation

**Contexte :**
Vérifier que l'authentification et l'autorisation fonctionnent correctement selon les spécifications.

**Scénario de test :**

**Test 1 : Connexion réussie**
```go
func TestLogin_Success(t *testing.T) {
    // Arrange
    user := createTestUser("user@example.com", "password123")
    
    // Act
    response := login(user.Email, user.Password)
    
    // Assert
    assert.Equal(t, 200, response.StatusCode)
    assert.NotEmpty(t, response.Token)
    assert.NotEmpty(t, response.RefreshToken)
}
```

**Résultat attendu :** Connexion réussie, tokens générés

**Test 2 : Connexion échouée (mauvais mot de passe)**
```go
func TestLogin_InvalidPassword(t *testing.T) {
    // Arrange
    user := createTestUser("user@example.com", "password123")
    
    // Act
    response := login(user.Email, "wrongpassword")
    
    // Assert
    assert.Equal(t, 401, response.StatusCode)
    assert.Empty(t, response.Token)
}
```

**Résultat attendu :** Erreur 401, pas de token

**Test 3 : Accès protégé sans authentification**
```go
func TestProtectedEndpoint_NoAuth(t *testing.T) {
    // Act
    response := get("/api/v1/clusters", nil)
    
    // Assert
    assert.Equal(t, 401, response.StatusCode)
}
```

**Résultat attendu :** Erreur 401, accès refusé

**Test 4 : Accès protégé avec authentification**
```go
func TestProtectedEndpoint_WithAuth(t *testing.T) {
    // Arrange
    token := loginAndGetToken("user@example.com", "password123")
    
    // Act
    response := get("/api/v1/clusters", token)
    
    // Assert
    assert.Equal(t, 200, response.StatusCode)
    assert.NotEmpty(t, response.Clusters)
}
```

**Résultat attendu :** Accès autorisé, données retournées

**Test 5 : Autorisation (rôle admin vs user)**
```go
func TestAuthorization_AdminOnly(t *testing.T) {
    // Arrange
    adminToken := loginAndGetToken("admin@example.com", "password123")
    userToken := loginAndGetToken("user@example.com", "password123")
    
    // Act
    adminResponse := delete("/api/v1/users/123", adminToken)
    userResponse := delete("/api/v1/users/123", userToken)
    
    // Assert
    assert.Equal(t, 200, adminResponse.StatusCode) // Admin autorisé
    assert.Equal(t, 403, userResponse.StatusCode)   // User non autorisé
}
```

**Résultat attendu :** Admin autorisé, user refusé (403)
