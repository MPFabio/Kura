# A2.1 : Cadrage du projet et formalisation des attendus

## 1. Cadrage du projet

### Définition des objectifs

**Objectif principal :**
Développer et déployer Kura, une plateforme DevOps unifiée permettant la gestion centralisée de Kubernetes, Terraform, Ansible et des pipelines CI/CD.

**Objectifs spécifiques :**

1. **Fonctionnels :**
   - Authentification centralisée (JWT, refresh tokens, rôles)
   - Gestion des clusters Kubernetes (visualisation, actions, terminal)
   - Gestion des états Terraform (upload, visualisation, synchronisation, drift)
   - Gestion des jobs Ansible (exécution, monitoring)
   - Gestion des pipelines CI/CD (visualisation, déclenchement)
   - Observabilité intégrée (métriques, logs, alertes)

2. **Non-fonctionnels :**
   - Performance : Temps de réponse API < 200ms (p95)
   - Disponibilité : 99.5%
   - Scalabilité : Support de 100+ clusters Kubernetes
   - Sécurité : Conformité RGPD, ISO 27001
   - Maintenabilité : Documentation complète, tests automatisés (couverture > 80%)

### Périmètre du projet

**Inclus :**
- Développement des services backend (Go)
- Développement du frontend (React + TypeScript)
- Configuration de l'infrastructure (Kubernetes, Kong, Kafka)
- Intégration avec les APIs externes (Kubernetes, Terraform Cloud, Ansible Tower)
- Tests automatisés (unitaires, intégration, e2e)
- Documentation (technique, utilisateur)

**Exclus (Phase 1) :**
- Gestion des coûts cloud
- Gestion des secrets (HashiCorp Vault)
- Gestion des certificats SSL/TLS automatisée
- Backup et restauration automatisés (hors base de données)

### Cadre réglementaire

**Conformité requise :**
- **RGPD :** Gestion des données personnelles (utilisateurs, logs)
  - Consentement, droit à l'oubli, portabilité
  - Chiffrement des données sensibles
  - Audit et traçabilité

- **ISO 27001 :** Sécurité de l'information
  - Gestion des risques
  - Contrôles de sécurité
  - Audit et certification (Phase 3)

- **Conformité cloud :**
  - AWS Well-Architected Framework
  - Azure Security Best Practices
  - GCP Security Best Practices

### Ressources matérielles et logistiques

**Infrastructure :**
- Environnements de développement (laptops)
- Accès aux clouds (AWS, Azure, GCP)
- Cluster Kubernetes (dev, staging, prod)
- Outils de développement (IDE, Git, CI/CD)

**Licences :**
- IDE (VS Code, IntelliJ IDEA)
- Outils de monitoring (Prometheus, Grafana - open-source)
- Outils de sécurité (Snyk, Dependabot)

#### Résumé du cahier des charges

**Contexte :**
Les équipes DevOps utilisent une multitude d'outils isolés (Lens, Terraform Cloud, Ansible Tower, Jenkins, Prometheus), ce qui génère une charge cognitive élevée et une difficulté à avoir une vision transversale.

**Solution :**
Plateforme DevOps unifiée (Kura) offrant une interface unique pour toutes les opérations DevOps, avec authentification centralisée, corrélation d'événements et observabilité intégrée.

**Besoins fonctionnels principaux :**
- Authentification centralisée (JWT, refresh tokens, rôles)
- Gestion des clusters Kubernetes (CRUD, actions, terminal)
- Gestion des états Terraform (upload, visualisation, synchronisation, drift)
- Gestion des jobs Ansible (exécution, monitoring)
- Gestion des pipelines CI/CD (visualisation, déclenchement)
- Observabilité (métriques, logs, alertes)

**Besoins non-fonctionnels :**
- Performance : Temps de réponse < 200ms (p95)
- Disponibilité : 99.5%
- Scalabilité : Support de 100+ clusters
- Sécurité : Conformité RGPD, ISO 27001
- Maintenabilité : Documentation, tests automatisés

### Spécifications techniques

#### Architecture

**Pattern :** Microservices
- Séparation des responsabilités par domaine
- Communication via API Gateway (Kong)
- Event-driven (Kafka)

**Technologies :**
- Backend : Go, Gin, Kubernetes client-go
- Frontend : React, TypeScript, Redux Toolkit
- Infrastructure : Kubernetes, Docker, Kong, Kafka
- Base de données : PostgreSQL, Redis
- Monitoring : Prometheus, Grafana

#### Spécifications fonctionnelles

**Authentification :**
- Inscription, connexion (JWT)
- Refresh tokens
- Gestion des rôles (admin, user)
- Middleware d'authentification

**Gestion Kubernetes :**
- Liste des clusters, namespaces, pods, deployments, services
- Actions (créer, modifier, supprimer)
- Terminal interactif (WebSocket)
- Monitoring (métriques, logs)

**Gestion Terraform :**
- Upload d'états Terraform
- Visualisation des ressources
- Synchronisation cloud (S3, Azure Blob, GCP Storage)
- Détection de drift (GCP, AWS, Azure)

**Gestion Ansible :**
- Exécution de jobs
- Monitoring des jobs
- Gestion des inventaires

**Gestion CI/CD :**
- Visualisation des pipelines
- Déclenchement de pipelines
- Monitoring des exécutions

**Observabilité :**
- Métriques (Prometheus)
- Logs centralisés
- Alertes (Prometheus Alertmanager)
- Dashboards (Grafana)

#### Spécifications non-fonctionnelles

**Performance :**
- Temps de réponse API < 200ms (p95)
- Temps de chargement page < 2s
- Support de 100+ clusters simultanés

**Disponibilité :**
- Uptime : 99.5%
- RTO : 1-4 heures selon composant
- RPO : 0-1 heure selon composant

**Scalabilité :**
- Auto-scaling horizontal (Kubernetes HPA)
- Cache (Redis) pour performance
- Load balancing (Kong, Kubernetes)

**Sécurité :**
- Authentification (JWT)
- Chiffrement (TLS en transit, au repos)
- Audit et traçabilité
- Conformité RGPD, ISO 27001

**Maintenabilité :**
- Documentation complète
- Tests automatisés (couverture > 80%)
- CI/CD (tests, déploiement)
- Code review

### Spécifications fonctionnelles détaillées

#### User Stories

**Authentification :**
- En tant qu'utilisateur, je veux m'inscrire pour accéder à la plateforme
- En tant qu'utilisateur, je veux me connecter avec mon email et mot de passe
- En tant qu'utilisateur, je veux rafraîchir mon token d'accès
- En tant qu'admin, je veux gérer les rôles des utilisateurs

**Gestion Kubernetes :**
- En tant qu'utilisateur, je veux voir la liste de mes clusters Kubernetes
- En tant qu'utilisateur, je veux voir les pods d'un namespace
- En tant qu'utilisateur, je veux créer un deployment
- En tant qu'utilisateur, je veux accéder à un terminal interactif

**Gestion Terraform :**
- En tant qu'utilisateur, je veux uploader un état Terraform
- En tant qu'utilisateur, je veux visualiser les ressources d'un état
- En tant qu'utilisateur, je veux synchroniser un état depuis le cloud
- En tant qu'utilisateur, je veux détecter le drift d'un état

**Gestion Ansible :**
- En tant qu'utilisateur, je veux exécuter un job Ansible
- En tant qu'utilisateur, je veux voir le statut d'un job
- En tant qu'utilisateur, je veux voir les logs d'un job

**Gestion CI/CD :**
- En tant qu'utilisateur, je veux voir la liste de mes pipelines
- En tant qu'utilisateur, je veux déclencher un pipeline
- En tant qu'utilisateur, je veux voir le statut d'une exécution

**Observabilité :**
- En tant qu'utilisateur, je veux voir les métriques d'un cluster
- En tant qu'utilisateur, je veux voir les logs d'un service
- En tant qu'admin, je veux configurer des alertes
