ModulOps - État du Projet et Roadmap

================================================================================
CONTEXTE DU PROJET
================================================================================

VISION

ModulOps est une plateforme DevOps unifiée qui centralise la gestion et l'observation de plusieurs outils DevOps couramment utilisés en entreprise : Kubernetes, Terraform, Ansible, pipelines CI/CD, métriques et alertes.

PROBLÉMATIQUE

Les équipes DevOps jonglent aujourd'hui entre plusieurs interfaces isolées :
- Portails Kubernetes (Lens, Octant, dashboards maison)
- Outils Terraform (Terraform Cloud, Atlantis)
- Interfaces Ansible Tower / AWX
- Outils CI/CD (GitHub Actions, GitLab CI, Jenkins)
- Stacks d'observabilité (Prometheus / Grafana)

Problèmes identifiés :
- Difficile d'avoir une vision transversale (ex. : "ce pipeline déploie quels clusters / quelles ressources Terraform ?")
- Authentification et rôles dupliqués dans chaque outil
- Charge cognitive élevée : plusieurs outils à maîtriser
- Diagnostics cross-systèmes complexes

POSITIONNEMENT VS CONCURRENTS

Backstage (Spotify), Port, Cortex :
- Excellents pour la découverte et la documentation
- Légers sur l'opérationnel actif (gestion K8s, exécution Terraform, jobs Ansible)
- Orientés "catalogue" plutôt que "console d'opération"

ModulOps :
- Focus sur l'opérationnel actif (exécution Terraform, gestion K8s, jobs Ansible)
- Architecture microservices native avec bus d'événements Kafka
- Console d'opération pour équipes Ops/DevOps

VALEUR AJOUTÉE

- Point d'entrée unique : un seul portail, une seule API Gateway
- Agrégation des infos clés au même endroit
- Authentification centralisée avec rôles homogènes
- Événements corrélés via Kafka (ex. : déploiement Terraform → métriques → alertes)
- Garde-fous globaux (rôles, droits, observabilité, audit) au niveau plateforme

================================================================================
ARCHITECTURE TECHNIQUE
================================================================================

STACK TECHNOLOGIQUE

Backend :
- Go 1.21+ : Services principaux (auth, k8s, terraform, pipeline, alert, metrics)
- Python 3.11+ : Service Ansible
- PostgreSQL 15+ : Base de données principale
- Redis 7+ : Cache distribué
- Kafka : Message broker pour événements asynchrones
- Kong 3.4 : API Gateway

Frontend :
- React 18 + TypeScript 5
- Vite : Build tool
- Material-UI (MUI) : Composants UI
- React Query : Gestion des données
- Socket.io Client : WebSocket pour temps réel
- Recharts : Visualisations

Infrastructure :
- Docker Compose : Développement local
- Kubernetes : Orchestration production
- Prometheus + Grafana : Observabilité

ARCHITECTURE MICROSERVICES

Frontend (React+TS) 
    ↓
API Gateway (Kong)
    ↓
Services Backend (Go/Python)
    ↓
Event Bus (Kafka) → Services d'alertes/métriques
    ↓
PostgreSQL / Redis / Prometheus
